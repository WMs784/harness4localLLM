# harness4localLLM

`harness4localLLM` は、ローカル環境で動作する大規模言語モデル（LLM）を安全に制御（ハーネス）し、ターミナル操作の自動化やナレッジベース（Zettelkasten / Obsidian等）の構築をサポートする、Python製の自律型CLIエージェントです。

Ollama経由でローカルLLMを呼び出し、厳格に制限されたLinuxコマンドを実行させながら、日々のメモ（Daily Note）から有機的に繋がる知識ネットワーク（Literature / Permanent Note）を自動生成します。

---

## 🚀 主な機能

- **安全なCLIコマンド制御（ハーネス）**:
  LLMに直接ターミナルを解放するのではなく、あらかじめ許可されたコマンド（`ls`, `pwd`, `cat`, `grep`, `touch`, `echo`, `curl`）のみを実行可能なサンドボックス環境。
- **Zettelkastenメソッドの自動化**:
  - **Literature Note ワークフロー**: デイリーノートに含まれるURLを検知し、中身を自動スクレイピング・日本語要約。重要なキーワードを `[[WikiLink]]` 形式に自動変換し、元ノートへのバックリンクも付与。
  - **Permanent Note ワークフロー**: 散らばった文献ノートやメモを深く分析・抽象化し、構造化された「概念ノート」として統合。
- **実戦的なWebスクレイピング＆HTMLクレンジング対策**:
  - **不要タグの完全除去**: cURLで取得した生HTMLから、最大のノイズとなる `<script>`（JavaScript）や `<style>`（CSS）タグを中身ごと自動で完全抹消。
  - **本文マーカーによる前方ノイズカット**: `<article>` や `<main>` などの主要なHTML構造タグ、あるいは特定のクラス・ID名（`class="article"`等）を検知し、ログインフォームやヘッダーメニューなどの不要な前方データを丸ごとカット。
  - **プレーンテキスト化と空白集約**: 残ったHTMLタグをすべて消去して「純粋な日本語テキスト」のみを抽出し、無駄な連続改行を圧縮。LLMへのインプットを極限まで軽量・最適化。
  - **動的サイト対応（Zenn等）**: ZennなどのSPA（シングルページアプリケーション）のURLを検知すると、自動で軽量なJSON APIへエンドポイントを切り替えて取得。
  - **文字コードの自動判定**: `UTF-8` だけでなく、国内ニュースサイト等に多い `Shift_JIS (CP932)` も自動でフォールバックしてデコード。
  - **User-Agent偽装**: cURL実行時に通常のブラウザのUser-Agentを自動挿入し、ボットブロックを回避。

## 📁 ディレクトリ構造

本ツールは以下のディレクトリ構造を前提（または自動生成）して動作します。Obsidianなどの保管庫（Vault）の直下に配置して運用することも可能です。

```text
.
├── cli_agent.py          # エージェント本体
├── README.md             # このファイル
├── Daily/                # 日々のメモ（*.md）を配置する場所
├── FleetingNote/         # 一時的なメモを配置する場所
├── LiteratureNote/       # 自動生成された文献ノートの出力先
└── PermanentNote/        # 自動統合された概念ノートの出力先
```

## 🛠️ セットアップ

### 1. 前提条件

- Python 3.10 以上
- [Ollama](https://ollama.com/) がローカルで起動していること

### 2. 必要ライブラリのインストール

```bash
pip install langchain-core langchain-ollama
```

### 3. ローカルLLMの準備

本スクリプトはデフォルトで `gemma4:e4b`（環境に合わせて要変更）を使用します。あらかじめモデルをプルしておいてください。

```bash
ollama pull gemma4:e4b
```

> 💡 コード内の `MODEL_NAME` 変数をお使いのローカルモデル（例: `gemma2` など）に変更することで、任意のモデルで動かすことができます。
>
> 🛠️ **長文コンテキストへの対応**: スクリプト内部でOllamaの記憶容量（コンテキストサイズ）をデフォルトの2,048トークンから **16,384トークン（`num_ctx=16384`）** へ物理的に拡張しています。これにより、長文記事も途中で破綻することなく安定して読解可能です。

## 💻 使い方

スクリプトを実行すると、専用のインタラクティブシェルが起動します。

```bash
python cli_agent.py
```

### 1. エージェントとの対話（チャットモード）

通常入力はローカルLLMとの対話になります。エージェントが必要と判断した場合、自動的にコマンドを実行して結果を報告します。

```text
CLI Agent v1.0
あなた > Dailyフォルダの中にどんなファイルがあるか確認して
EXECUTE: ls Daily
2026-06-09.md  2026-06-13.md
Dailyフォルダ内には「2026-06-09.md」と「2026-06-13.md」が存在しています。
```

### 2. 組み込みワークフローの実行

`/workflow:` から始まるコマンドを入力することで、ノート構築タスクを全自動で実行します。

| コマンド                                      | 説明                                                                                 |
| :-------------------------------------------- | :----------------------------------------------------------------------------------- |
| `/workflow:create_literature_note_from_daily` | `Daily/` 内のノートからURLを抽出し、`LiteratureNote/` を自動生成します。             |
| `/workflow:draft_permanent_note`              | `FleetingNote/` と `LiteratureNote/` の中身を統合し、`PermanentNote/` を生成します。 |
| `/workflow:test`                              | 各ノートフォルダの存在チェックを行います。                                           |
| `/workflow:help`                              | 利用可能なワークフローの一覧を表示します。                                           |

## 🔒 セキュリティ & 制約事項

- **ハルシネーション（嘘）の強力な抑制**:
  システムプロンプトにより、エージェントが実行していないコマンドの結果や、Webページ・ファイルの内容を「推測」「捏造」することを厳格に禁止しています。
- **コマンドのホワイトリスト化**:
  `ALLOWED_COMMANDS` に登録されていない未知のコマンド（例: `rm`, `mv` や任意のスクリプトなど）をLLMが生成した場合、実行前にプログラム側で遮断（`Denied`）します。
- **「Lost in the Middle」現象の回避（最末尾命令）**:
  長文の本文を読み進めるうちにLLMがルールを忘れてしまう現象を防ぐため、文献ノート要約時の「`[[WikiLink]]`（2重角括弧）化のルール」等の厳格な制約事項は、**プロンプトの最末尾（本文の直後）に配置・念押し**する堅牢なプロンプト構造を採用しています。
